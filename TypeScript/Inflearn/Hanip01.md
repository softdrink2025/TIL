## 타입스크립트란?
- 자바스크립트의 확장판
- 자바스크립트를 더 안전하게 사용할 수 있도록 타입 관련 기능들을 추가한 언어
- Node.js의 등장으로 웹브라우저에서만 실행될 수 있었던 자바스크립트가 어디서든 구동이 가능해졌다.
- 기존의 자바스크립트의 유연한 문법이 버그 발생 가능성을 높이기 때문에 복잡한 프로그램을 만들 시 전체적인 안정성이 떨어짐
- 이를 극복하기 위해 안정성을 추가로 확보한 언어의 필요성 대두 => TypeScript 탄생

## 타입 시스템
- 동적 타입 시스템
    - 코드를 실행하면서 유동적으로 변수의 타입을 결정 (ex JavaScript)
    - 코드의 타입 오류를 미리 검사할 수 없기 때문에 예기치 못한 오류가 발생할 수 있다.

- 정적 타입 시스템
    - 코드를 실행하기 전에 정적으로 변수의 타입을 결정(ex Java)
    - 모든 변수에 일일이 타입을 지정해주어야 하기 때문에 타이핑 양이 증가함, 유연하지 못함

- 점진적 타입 시스템(Gradual Type System) 
    - 모든 변수에 타입을 일일이 지정할 필요 없음
    - 실행 전 검사를 통한 타입 안정성 확보
    - 자동으로 변수의 타입을 추론함
